1) Python 自动化核心包

你要掌握到什么程度

  • 能写脚本：抓网页/API、处理 JSON/CSV/Excel、写文件、写日志、异常重试、定时执行。
  • 能看懂并改 AI 生成代码：尤其是 requests / bs4、数据结构、函数、错误栈。

基础部分（CS50P 已覆盖）

如果你已经学完 CS50P（至少 L1-L6），下面这些基础教程不再作为必学任务：

  • Python 官方教程里基础语法章节（Control Flow / Functions / Modules / Input/Output / Exceptions）。
  • Automate the Boring Stuff with Python、freeCodeCamp Python、PY4E 的入门部分。

保留：直接支持项目的文档

logging / csv / json

  • basicConfig、logger、日志级别、写入文件。
  • json.load / json.dump
  • csv.DictReader / csv.DictWriter

写脚本必备库文档（学会查，不用背）

  • requests Quickstart：会话、超时、headers。
  • BeautifulSoup 官方文档：find / find_all、属性选择、文本提取。
  • Real Python 的 bs4 实战：看一遍形成“从 0 到 1”骨架。

CS50P 基础已覆盖，重点投入实战文档和后续模块。

补强模块：pytest / Selenium

pytest（接口与脚本测试主力）

  • 先掌握：测试发现、断言、fixture、parametrize、mark。
  • 目录建议：tests/api、tests/ui、conftest.py。
  • 目标：能把你的核心脚本改成“可回归”的自动化测试。

Selenium（只做关键流程 UI 冒烟）

  • 重点不是全覆盖，而是登录、查询、提交这类关键路径。
  • 必须会：显式等待、稳定定位器、失败截图。
  • 目标：能解释“为什么 UI 自动化要少而精”。

2) HTTP / JSON / 状态码 / 合规爬取

必学要点

  • HTTP：GET/POST、headers、状态码（200/403/404/429/5xx）、超时与重试。
  • JSON：把它当“字典/列表嵌套”处理。
  • 合规：优先 RSS/API，尊重 robots.txt，控制频率，不碰登录/付费墙/隐私数据。

推荐资源

  • MDN：HTTP 概览（请求-响应、状态码）。
  • MDN：JSON 基础（读懂结构）。
  • Robots Exclusion Protocol（RFC 9309）：知道 robots 是什么以及为何要尊重。
  • Automate the Boring Stuff 爬虫章节：优先 API、结构会变、DevTools 定位元素。

接口自动化与 API 测试框架

  • 框架建议：pytest + requests 作为主线，先稳再扩展。
  • 用例结构：按“鉴权 / 查询 / 创建 / 更新 / 删除 / 异常码”分层组织。
  • 断言分层：状态码、响应字段、业务规则（而不只断言 200）。
  • 数据治理：测试数据可回收、环境隔离（dev/test/prod 分离）、避免脏数据污染。
  • 接口契约：关键接口可加 JSON Schema 或字段完整性校验。

接口自动化资源（按顺序）

  • pytest 官方文档：fixture、参数化、标记、测试组织。
  • requests Quickstart：会话、headers、timeout、异常。
  • JSON Schema：接口返回结构校验思路。
  • Selenium with Python：定位器、等待、页面交互。

进阶方向

  • JS 逆向、复杂反爬对抗、代理池工程化。
  • Scrapy（除非岗位明确要求）。

3) OpenClaw / Clawbot

目标：能装、能跑、能接渠道、能解释架构、能说安全边界。

最小知识图谱

  • Onboarding / 安装与基础运行：Node 版本要求、openclaw onboard 流程、Gateway 验证。
  • Gateway / CLI 排障：doctor、迁移/状态检查、日志定位思路。
  • Channels：Telegram 快速落地、WhatsApp 复杂性、allowlist/DM/群组差异。
  • Skills：目录结构、SKILL.md（含 YAML frontmatter）、加载/过滤机制。
  • Sandboxing：Docker 可降低 blast radius，但不是完美边界；默认不开 sandbox 的风险要能讲清。

官方学习入口（顺序）

  1. Docs 首页（当目录）
  2. Channels 文档
  3. Skills 文档
  4. Doctor 文档
  5. Sandboxing 文档
  6. README（CLI 与架构线索）

安全意识

要能主动说出安全落地原则：最小权限、环境隔离、密钥管理、技能审查、操作审计。

4) n8n 工作流编排

典型落地链路：定时触发 → HTTP/API → 处理 → 推送 → 出错告警。

核心节点

  1. Schedule Trigger：定时任务。
  2. HTTP Request：超时、分页、鉴权。
  3. Expressions：取字段、拼接、日期处理、基础清洗。
  4. Code Node：必要时写少量 JS/Python 做定制处理。
  5. Error handling / Error Trigger：上线流程必须有错误链路。

资源与范围

  • 官方 Text courses：按需挑主题学习。
  • 项目驱动练习（n8n workflows）：做 2-3 个端到端工作流即可，不要陷入超长刷课。

进阶方向

  • 自定义 Node、n8n 内核、复杂自托管架构。
  • 大量 JS 表达式技巧（按需补充，靠搜索补齐）。

安全提醒

控制编辑权限、持续升级版本、生产环境隔离，尤其是涉及 Code Node 的流程。

5) LLM API 接入

目标不是训练模型，而是把“采集”变成“可结构化处理的 AI 自动化”。

核心能力

  • 能调用至少一家 API（OpenAI / Claude / Qwen）。
  • 能控制成本（截断上下文、只传关键段落）。
  • 能输出结构化 JSON，方便入库与推送。
  • 理解 function calling / tool use 的基本概念。

OpenAI（如用 OpenAI 生态）

  • Developer Quickstart：先跑通第一条请求。
  • Python SDK：知道如何发请求与取结果。
  • Structured Outputs（重点）：json_schema 或 function calling 严格 JSON。
  • Responses API 迁移：了解即可。
  • 旧式 Completions（除非维护遗留项目）。
  • 深入流式 / Realtime（短期非刚需）。

Claude / Qwen

  • Claude：Build with Claude 入口 与 Tool use 概念。
  • Qwen：DashScope / Model Studio API 参考，跑通一条即可。
  • DashScope API 平台：了解 API 形态和参数风格。

6) 定时运行 + 部署 + 运维

不必上 K8s，但要能说明“怎么稳定跑、失败怎么发现”。

必须会的 3 件事

  1. 定时运行：cron 或 GitHub Actions schedule（二选一）。
  2. 日志与告警：logging + 失败推送（Telegram/邮件）或 n8n Error workflow。
  3. 配置与密钥：.env / 环境变量，不把 key 写死，也不提交 .env。

CI 细节

  • 触发策略：至少配置 pull_request + push(main) + 可选 schedule。
  • 标准步骤：checkout -> setup-python -> install deps -> run pytest。
  • 失败可追踪：失败时上传日志/测试报告（artifact），便于复盘。
  • 合并门槛：开启 required checks，CI 红灯不允许合并。

资源

  • crontab 五段式语法（每天/每小时任务会写）。
  • GitHub Actions schedule（UTC、最小间隔等）。
  • GitHub Actions：Python 构建与测试（官方 workflow 模板）。
  • Branch protection / required checks（把 CI 结果变成合并门槛）。
  • python-dotenv 与 Twelve-Factor Config 原则。
  • Docker 入门（用于 sandboxing 或脚本打包部署）。
  • CSDIY：MIT Missing Semester（Shell/Git/SSH/tmux 工具链）；可配合 官方课程 与 中文站快速补齐工程效率。

进阶方向

  • Kubernetes、复杂 CI/CD、服务网格。
  • 复杂监控栈（Prometheus/Grafana）。

进阶方向

9) Git 与交付

交付可维护系统、快速上手、降低管理成本是核心目标。

必须会

  • init / clone / add / commit / push
  • 查看历史、简单回滚、远程协作。
  • 分支提交流程：能在 PR 中解释改动、修复评论、等 CI 通过再合并。

交付证据包

  • 1 页简历，突出核心能力与项目结果指标。
  • 可运行仓库 + README（新电脑 10 分钟内跑通）。
  • 1 分钟录屏，展示完整闭环，不只展示成功截图。
  • 日志与告警截图 + 1 条排障记录（问题-原因-修复-验证）。
  • 项目结果量化：吞吐量、失败率、重试成功率、节省人时至少 3 项。

快速上手计划

  • 首周交付模板：D1 环境与权限，D3 跑通闭环并接入告警，D5 补齐稳定性并交付 Runbook。
  • 排障按"复现-定位-修复-回归"四步法系统处理。

学习要点

  • 动手写代码，不只停留在概念记忆，要有可运行 Demo 和结果指标。
  • 掌握完整链路：成功路径 + 失败分支 + 修复过程。
  • 每个模块的项目结果要能量化。

资源

  • CSDIY：Git 学习路径（常用 Git 能力按路线梳理）。
  • Pro Git：2.1 / 2.2 / 2.3 / 2.4 / 2.5（建库、提交、历史、撤销、远程）。
  • Learn Git Branching：可视化练习分支、合并与回滚。
  • How to Write a Git Commit Message：规范提交信息，提升交付表达。
  • GitHub Docs：Git basics（按步骤走一遍）。

进阶方向

  • 复杂 rebase、深度分支策略、Git hook（短期不值）。

10) 技能掌握检查表

按 1-9 模块逐项自测，检验学习成果。

A. Python 自动化核心包

  • 我能写出含 requests + bs4 + try/except + logging 的可运行脚本。
  • 我能用 pytest 写基础测试（fixture、参数化、断言）。
  • 我能用 Selenium 做关键流程冒烟，并处理等待与截图。

B. HTTP / JSON / 合规爬取

  • 我能解释并处理 200/403/429/5xx，不只看 200。
  • 我能写接口自动化用例，覆盖状态码、字段、业务规则三层断言。
  • 我知道 robots、ToS、频率控制和数据边界，不碰高风险采集。

C. OpenClaw / Clawbot

  • 我能讲清 Gateway/Channels/Skills，并跑通一个渠道。
  • 我会用 doctor / status / 日志做排障。
  • 我能说明 sandboxing 的价值和限制。

D. n8n 工作流编排

  • 我会 Schedule Trigger、HTTP Request、Expressions、Error workflow。
  • 我能把失败分支显式接入告警而不是静默失败。

E. LLM API 接入

  • 我能跑通至少一家 LLM API 并输出结构化 JSON。
  • 我能解释如何做成本控制、降级与不确定性标记。

F. 定时运行 / 部署 / 运维

  • 我能用 cron 或 GitHub Actions schedule 定时执行任务。
  • 我会配置 pull_request + push 自动跑 pytest。
  • 我会看 CI 失败日志/artifact 并用 required checks 控制合并风险。

G. 数据处理

  • 我能把数据落到 CSV/Excel/SQLite，并做去重与字段规范化。
  • 我能解释数据质量校验和增量更新策略。

H. 推送与对接

  • 我能把日报和失败告警推送到 Telegram 或 Email。
  • 我能区分业务通知与错误告警，避免信息噪音。

I. Git 与交付

  • 我有完整证据包：1 页简历、可运行仓库、README、录屏、日志与排障记录。
  • 我能用量化指标讲项目结果，不只描述过程。
  • 我能说出首周交付计划（D1/D3/D5）并给出明确到岗信号。
  • 我能讲清失败分支与修复过程，不只展示成功路径。
  • 聚焦高度匹配的岗位方向：Python 自动化 / 接口自动化 / 运维自动化。
